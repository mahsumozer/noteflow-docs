Tools#

Noteflow MCP provides 25 tools. All of them can be selected via NOTEFLOW_FEATURES.

Notes (notes:read, notes:write)#

You can read and manage all note types: text, canvas, voice, image, drawing, presentation, page, and calendar.

What you can do:

  • List and filter your recent notes

  • Search by title, content, or tag

  • Create a new note, edit an existing one, or move it to trash

Example prompts:

“Show the notes I created this week” “List all my notes tagged ‘idea’” “Create a new note about API integration”

Projects (projects:read, projects:write)#

Access to kanban-style project boards.

What you can do:

  • List your projects

  • View project details and columns

  • Create and update projects

Example prompts:

“Show my active projects” “Get the details of the ‘Mobile App’ project”

Folders (folders:read, folders:write)#

Folder hierarchy within a project.

What you can do:

  • List folders and view their contents

  • Create a new folder

Tags (tags:read)#

What you can do:

  • List all tags

  • Find notes with a specific tag

Example prompts:

“What tags do I have?” “List my notes tagged ‘design’”

Tasks (tasks:read, tasks:write)#

Task cards on project boards.

What you can do:

  • List tasks and view their details

  • Create a new task

  • Move a task to a different column

Example prompts:

“Show the tasks in the To Do column” “Move the ‘Backend fix’ task to the Done column”

AI (ai)#

Use Noteflow’s AI features.

What you can do:

  • Get an AI summary of a note

  • Perform a semantic similarity search (meaning-based, not keyword-based)

Example prompts:

“Summarize this note” “Find my notes related to ‘async programming’”

Calendar (calendar:read, calendar:write)#

Events inside calendar-type notes.

What you can do:

  • List events in calendar notes

  • Add a new event

Voice Notes (voice:read)#

What you can do:

  • List voice recordings

  • Read the content of notes that have a transcript

Example prompts:

“List my voice notes” “Find the voice recording where I talked about API design”
